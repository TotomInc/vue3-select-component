Vue 3 Select Component is an accessible select for Vue 3. It focuses on developer experience, type safety, and ease of use.

## What is Vue 3 Select Component?

v1 has two APIs:

- **Assembled `Select`**: drop-in component with familiar DX from earlier versions.
- **Primitives**: composable building blocks (`SelectRoot`, `SelectTrigger`, `SelectPopover`, and more) for custom UX.

Both APIs share the same state machine, accessibility behavior, and TypeScript types.

## What you can do

- **Bind data with `v-model`**: single or multi-select with two-way binding
- **Search and filter options**: built-in search with custom filter logic
- **Customize rendering**: primitive slots for options, values, tags, and indicators
- **Style your way**: default assembled styles with `--vs-*` variables, or unstyled primitives with `data-*` hooks
- **Teleport the menu**: render the dropdown anywhere in the DOM

## When to use it

Use the **assembled `Select`** for a production-ready select with sensible defaults. Good for migrations from `vue-select` or Vue 2.

Use **primitives** when you need custom UX like infinite scroll, custom positioning, or non-standard layouts.

::note
This documentation targets the v1 API in development. See the [migration guide](/guide/migration) if you are upgrading from v0.
::
