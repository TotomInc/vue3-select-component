Primitives are headless building blocks that share state via `SelectRoot`. Each one handles a single concern like trigger, popover, or listbox.

## Understand the architecture

```
SelectRoot (state + context provider)
├── SelectTrigger
│   ├── SelectIcon
│   ├── SelectValue
│   ├── SelectInput (when searchable)
│   ├── SelectTrailingIcon
│   └── SelectClear
└── SelectPopover
    └── SelectListbox
        ├── SelectNoOptions
        └── SelectOption (×N)
            └── optional: SelectGroup, SelectGroupLabel, SelectSeparator
```

Cross-component state uses provide/inject. Primitives must live inside `SelectRoot`. Group primitives are optional. Use them only when you need labelled sections or separators in the list.

## Build a searchable single select

```vue [CustomSelect.vue]
<script setup lang="ts">
import { ref } from "vue"
import {
  SelectRoot,
  SelectTrigger,
  SelectValue,
  SelectIcon,
  SelectTrailingIcon,
  SelectClear,
  SelectPopover,
  SelectInput,
  SelectListbox,
  SelectOption,
  SelectNoOptions,
} from "vue3-select-component/primitives"

const options = [
  { value: "js", label: "JavaScript" },
  { value: "ts", label: "TypeScript" },
]

const model = ref<string | null>(null)
</script>

<template>
  <SelectRoot v-model="model" :options="options" searchable clearable>
    <SelectTrigger>
      <SelectIcon>
        <UIcon name="i-lucide-code" />
      </SelectIcon>
      <SelectValue placeholder="Pick a language" />
      <SelectInput />
      <SelectTrailingIcon />
      <SelectClear />
    </SelectTrigger>

    <SelectPopover>
      <SelectListbox>
        <SelectNoOptions>No results</SelectNoOptions>
        <SelectOption
          v-for="opt in options"
          :key="opt.value"
          :value="opt.value"
          :label="opt.label"
        />
      </SelectListbox>
    </SelectPopover>
  </SelectRoot>
</template>
```

## Register options two ways

**Via `options` prop on `SelectRoot`**: pass a normalized array:

```ts
{ label: string; value: string | number; disabled?: boolean }[]
```

**Via declarative `SelectOption` children**: register options on mount. Both sources merge. Declarative options are skipped when their `value` already exists from the prop.

## Group options

::note
You do not need group primitives for a flat list. Render `SelectOption` children directly inside `SelectListbox`, as in the example above.
::

Reach for `SelectGroup`, `SelectGroupLabel`, and `SelectSeparator` only when options belong in labelled sections or need visual dividers. Groups and separators are not selectable and do not change keyboard navigation.

```vue
<SelectListbox>
  <SelectGroup>
    <SelectGroupLabel>Frontend</SelectGroupLabel>
    <SelectOption value="vue" label="Vue" />
    <SelectOption value="react" label="React" />
  </SelectGroup>

  <SelectSeparator />

  <SelectGroup>
    <SelectGroupLabel>Backend</SelectGroupLabel>
    <SelectOption value="node" label="Node.js" />
    <SelectOption value="rust" label="Rust" />
  </SelectGroup>
</SelectListbox>
```

Grouped options still register as normal options. When filtering hides every option in a group, the group label is hidden too.

## Teleport the popover

`SelectPopover` teleports to `document.body` by default so the dropdown escapes overflow and stacking-context constraints. Scoped styles from parent components still apply to slot content after teleport.

Disable teleport or target a custom container with the `teleport` prop:

```vue
<!-- default: teleports to document.body -->
<SelectPopover>
  <SelectListbox />
</SelectPopover>

<SelectPopover teleport="#menu-portal">
  <!-- custom portal target -->
</SelectPopover>

<SelectPopover :teleport="false">
  <!-- render inline -->
</SelectPopover>
```

## Popover behavior

`SelectPopover` wraps reka-ui `Popover`. Pass reka-ui popover props to control placement, collision behavior, modality, and portal deferral:

```vue
<SelectPopover
  side="top"
  :side-offset="8"
  align="end"
  :collision-padding="16"
>
  <SelectListbox />
</SelectPopover>
```

The assembled `Select` accepts the same props and forwards them to `SelectPopover`. See [Popover props](/api/props#popover-props) for the full list.

## Customize with slots

Primitives expose slots for rendering. See [Slots](/api/slots) for the full reference.

| Primitive | Slot | Purpose |
|-----------|------|---------|
| `SelectValue` | `default` | Custom single/multi value display |
| `SelectOption` | `default` | Custom option content |
| `SelectGroup` | `default` | Group related options |
| `SelectGroupLabel` | `default` | Label a group for assistive tech |
| `SelectTag` | `default`, `remove` | Custom multi-value tags |
| `SelectIcon` | `default` | Leading icon before the value |
| `SelectTrailingIcon` | `default` | Custom trailing icon or loading UI |
| `SelectNoOptions` | `default` | Custom empty state |

## When to use primitives vs assembled Select

| Use assembled `Select` | Use primitives |
|------------------------|----------------|
| Standard dropdown UX | Custom layouts or infinite scroll |
| Quick integration | Non-standard trigger or popover behavior |
| Migrating from v0 | Full control over every DOM node |

See the [components reference](/api/components) for props and behavior of each primitive.
