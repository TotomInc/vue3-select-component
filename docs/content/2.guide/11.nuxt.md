Vue 3 Select Component works in Nuxt 3 and 4. No dedicated module is required.

## Install

Install the package as usual. See [Installation](/getting-started/installation).

## Import styles

Import the stylesheet once in `nuxt.config.ts`:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  css: ["vue3-select-component/styles"],
})
```

## Use in pages and components

Bind `v-model` the same way as in a Vite SPA:

```vue [pages/index.vue]
<script setup lang="ts">
import { ref } from "vue"
import Select from "vue3-select-component"

const selected = ref<string | null>(null)

const options = [
  { label: "Option #1", value: "option_1" },
  { label: "Option #2", value: "option_2" },
]
</script>

<template>
  <Select
    v-model="selected"
    :options="options"
    placeholder="Select an option"
  />
</template>
```

Primitives work the same way. Import from `vue3-select-component/primitives`.

## SSR and SSG caveats

The select teleports its dropdown to `document.body` by default. That portal is not present during server render and can cause hydration warnings.

Pick one approach:

### `ClientOnly` (recommended)

Wrap the component with [`ClientOnly`](https://nuxt.com/docs/4.x/api/components/client-only) for the simplest fix.

The control renders on the client only, once hydration is complete. Add a `#fallback` slot if you want to avoid layout shift.

```vue [pages/index.vue]
<template>
  <ClientOnly fallback="Loading...">
    <Select v-model="selected" :options="options" />
  </ClientOnly>
</template>
```

::note
This is what we use to write interactive docs demos in this project.
::

### Disable teleport

Pass `:teleport="false"` to keep the dropdown in the component tree. The closed trigger hydrates without a portal mismatch. Use this when you need the control in static HTML or inside overflow containers you control.

```vue [pages/index.vue]
<Select
  v-model="selected"
  :options="options"
  :teleport="false"
/>
```

With primitives, set `teleport` on `SelectPopover`.

### Defer portal

Pass `defer` on `Select` or `SelectPopover` to delay teleport until after mount. Keeps default body teleport without wrapping the whole control.

Learn more about [deferred teleports](https://vuejs.org/guide/built-ins/teleport.html#deferred-teleport) and [on Reka-UI popover portal docs](https://reka-ui.com/docs/components/popover#portal).

## Static generation

`nuxt generate` follows the same rules. Use `ClientOnly`, `:teleport="false"`, or `defer` on every page that renders a select. Search and keyboard behavior run after client hydration.

::note
Form attributes (`name`, `id`, `required`, and ARIA props) pass through to the trigger. They work in SSR markup when the control itself is server-rendered.
::
