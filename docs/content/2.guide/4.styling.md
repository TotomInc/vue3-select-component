## Default styles for assembled Select

The assembled `Select` ships optional default styles. Import them once in your app entry:

```ts
import "vue3-select-component/styles"
```

Primitives remain **unstyled** unless you import the stylesheet and target their `data-*` hooks yourself.

::note
Styles are not auto-injected (CSP-friendly). Import them manually, as in v0.
::

## CSS variables

Default styles use `--vs-*` variables (same prefix as v0). They are defined on `:root`, `[data-assembled-select]`, and `[data-select-popover]` so teleported dropdown panels inherit your theme.

Override variables on the assembled select root:

```vue [App.vue]
<script setup lang="ts">
import { ref } from "vue"
import Select from "vue3-select-component"
import "vue3-select-component/styles"

const value = ref<string | null>(null)

const options = [
  { label: "Option #1", value: "option_1" },
  { label: "Option #2", value: "option_2" },
]
</script>

<template>
  <Select v-model="value" :options="options" class="brand-select" />
</template>

<style scoped>
.brand-select {
  --vs-border-radius: 12px;
  --vs-outline-color: #0ea5e9;
}
</style>
```

Because the popover teleports to `document.body`, set popover-specific variables on `:root` or `[data-select-popover]` when needed:

```css
:root {
  --vs-menu-box-shadow: 0 8px 24px rgb(15 23 42 / 0.12);
}
```

## Headless styling with data attributes

Every primitive exposes stable `data-*` hooks for custom CSS:

| Attribute | Element | State |
|-----------|---------|-------|
| `data-select-root` | Root wrapper | |
| `data-assembled-select` | Assembled root | |
| `data-select-trigger` | Trigger button | `aria-expanded` |
| `data-select-value` | Value display | `data-empty` |
| `data-select-input` | Search input | |
| `data-select-popover` | Dropdown panel | `aria-hidden` |
| `data-select-listbox` | Listbox container | |
| `data-select-option` | Option row | `data-value`, `data-active` |
| `data-select-tag` | Multi-value tag | |
| `data-select-tag-remove` | Tag remove button | |
| `data-select-clear` | Clear button | |
| `data-select-icon` | Leading icon | |
| `data-select-trailing-icon` | Trailing icon / loader | `data-open`, `data-loading` |
| `data-select-no-options` | Empty state | |
| `data-select-create-item` | Create action | `data-active` |

### Style the trigger

```css
[data-select-trigger] {
  display: flex;
  align-items: center;
  gap: 0.5rem;
  width: 100%;
  min-height: 2.375rem;
  padding: 0.25rem 0.5rem;
  border: 1px solid #e4e4e7;
  border-radius: 0.375rem;
  background: #fff;
  cursor: pointer;
}

[data-select-trigger]:disabled {
  opacity: 0.5;
  cursor: not-allowed;
}
```

### Style options

Use `data-active` for keyboard-highlighted options and `aria-selected` for selected state:

```css
[data-select-option] {
  padding: 0.5rem 0.75rem;
  cursor: pointer;
}

[data-select-option][data-active="true"] {
  background: #dbeafe;
}

[data-select-option][aria-selected="true"] {
  background: #93c5fd;
}

[data-select-option][aria-disabled="true"] {
  opacity: 0.5;
  cursor: not-allowed;
}
```

### Style the popover

`SelectPopover` teleports to `document.body` by default. Use global CSS or unscoped styles with `[data-select-*]` hooks. Vue preserves scoped attribute selectors on teleported slot content, but parent `overflow` / `z-index` no longer constrain the panel.

```css
[data-select-popover] {
  margin-top: 0.5rem;
  border: 1px solid #e4e4e7;
  border-radius: 0.375rem;
  background: #fff;
  box-shadow: 0 1px 2px rgb(0 0 0 / 0.05);
  z-index: 50;
}
```

### Style multi-value tags

```css
[data-select-tag] {
  display: inline-flex;
  align-items: center;
  gap: 0.25rem;
  padding: 0.125rem 0.5rem;
  border-radius: 0.25rem;
  background: #f4f4f5;
  font-size: 0.75rem;
}

[data-select-tag-remove] {
  border: none;
  background: transparent;
  cursor: pointer;
  line-height: 1;
}
```

### Style the trailing icon

```css
[data-select-trailing-icon][data-open="true"] {
  transform: rotate(180deg);
}

[data-select-trailing-icon][data-loading="true"]::after {
  content: "";
  display: inline-block;
  width: 1rem;
  height: 1rem;
  border: 2px solid currentColor;
  border-top-color: transparent;
  border-radius: 50%;
  animation: spin 0.6s linear infinite;
}
```

## Scope styles to a wrapper

Wrap your select in a container class to scope overrides:

```vue [App.vue]
<script setup lang="ts">
import { ref } from "vue"
import Select from "vue3-select-component"
import "vue3-select-component/styles"

const value = ref<string | null>(null)

const options = [
  { label: "Option #1", value: "option_1" },
  { label: "Option #2", value: "option_2" },
]
</script>

<template>
  <div class="my-select">
    <Select v-model="value" :options="options" />
  </div>
</template>

<style scoped>
.my-select {
  --vs-border-radius: 12px;
}
</style>
```

For primitives, scope via a parent class or use `:deep()` as needed.
