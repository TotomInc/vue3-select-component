Vue 3 Select Component is built with TypeScript. Types flow from your option data through `v-model`, props, events, and slots so you get autocompletion and compile-time checks in both the assembled `Select` and primitive APIs.

## Generic components

`Select` and `SelectRoot` use [Vue 3.3 generic components](https://vuejs.org/api/sfc-script-setup.html#generics). Two type parameters drive inference:

| Parameter | Default | Role |
|-----------|---------|------|
| `OptionValue` | `string` | The type bound to `v-model` (`string` or `number`) |
| `GenericOption` | `SelectOptionData<OptionValue>` | Your source option objects |

```ts
// Assembled Select: OptionValue first, then GenericOption
Select<OptionValue, GenericOption>

// Primitives: OptionValue only
SelectRoot<OptionValue>
```

TypeScript usually infers both parameters from `:options` and `v-model`. Pass them explicitly when wrapping the component or when inference is ambiguous.

## Use the SelectOptionData type

The canonical option shape is exported as `SelectOptionData` (alias of internal `SelectOption`):

```ts
import type { SelectOptionData } from "vue3-select-component"

type SelectOptionData<Value extends string | number = string> = {
  label: string
  value: Value
  disabled?: boolean
}
```

::note
v0 exported this type as `Option<T>`. In v1, import `SelectOptionData` instead. See the [migration guide](/guide/migration#update-typescript-imports).
::

## Configure value types

`option.value` and `v-model` share the same type. Use `string` or `number`:

```vue [App.vue]
<script setup lang="ts">
import type { SelectOptionData } from "vue3-select-component"
import { ref } from "vue"
import Select from "vue3-select-component"

type UserOption = SelectOptionData<number>

const selectedUser = ref<number | null>(null)

const userOptions: UserOption[] = [
  { label: "Alice", value: 1 },
  { label: "Bob", value: 2 },
  { label: "Charlie", value: 3 },
  // { label: "David", value: "invalid" } // Type error
]
</script>

<template>
  <Select
    v-model="selectedUser"
    :options="userOptions"
    placeholder="Pick a user"
  />
</template>
```

## Extend option properties

Add custom fields and keep them typed in mappers, filters, and events:

```vue [App.vue]
<script setup lang="ts">
import type { SelectOptionData } from "vue3-select-component"
import { ref } from "vue"
import Select from "vue3-select-component"

type UserOption = SelectOptionData<number> & { username: string }

const selectedUser = ref<number | null>(null)

const userOptions: UserOption[] = [
  { label: "Alice", value: 1, username: "alice15" },
  { label: "Bob", value: 2, username: "bob01" },
  { label: "Charlie", value: 3, username: "charlie20" },
]
</script>

<template>
  <Select
    v-model="selectedUser"
    :options="userOptions"
    :get-option-label="(option) => `${option.label} (${option.username})`"
    placeholder="Pick a user"
  />
</template>
```

With primitives, the same `UserOption` type flows into `SelectOption` slot props and `@option-selected` payloads.

## Validate v-model types

The component enforces consistency between `option.value` and `v-model`:

```vue
<script setup lang="ts">
import type { SelectOptionData } from "vue3-select-component"
import { ref } from "vue"
import Select from "vue3-select-component"

type UserOption = SelectOptionData<number>

// Incorrect: should be ref<number | null>(null)
const selectedUser = ref<string | null>(null)

const userOptions: UserOption[] = [
  { label: "Alice", value: 1 },
  { label: "Bob", value: 2 },
]
</script>

<template>
  <!-- Type error: string conflicts with number -->
  <Select
    v-model="selectedUser"
    :options="userOptions"
    placeholder="Pick a user"
  />
</template>
```

## Use SelectModelValue

The internal `v-model` type is `SelectModelValue<Value>`:

```ts
import type { SelectModelValue } from "vue3-select-component"

type SelectModelValue<Value> = Value | Value[] | null
```

In practice, bind a narrower type for each mode:

| Mode | Bind as |
|------|---------|
| Single select | `ref<Value \| null>(null)` |
| Multi-select | `ref<Value[]>([])` |

```vue [App.vue]
<script setup lang="ts">
import type { SelectOptionData } from "vue3-select-component"
import { ref } from "vue"
import Select from "vue3-select-component"

type UserOption = SelectOptionData<number>

const single = ref<number | null>(null)
const multi = ref<number[]>([])

const userOptions: UserOption[] = [
  { label: "Alice", value: 1 },
  { label: "Bob", value: 2 },
]
</script>

<template>
  <Select v-model="single" :options="userOptions" />
  <Select v-model="multi" :options="userOptions" multiple />
</template>
```

## Type multi-select refs

When wrapping `Select`, pass both generics so events and options stay typed:

```vue [App.vue]
<script setup lang="ts">
import type { SelectOptionData } from "vue3-select-component"
import { ref } from "vue"
import Select from "vue3-select-component"

type UserOption = SelectOptionData<number>

const selectRef = ref<InstanceType<typeof Select<number, UserOption>>>()
const selectedUsers = ref<number[]>([])

const userOptions: UserOption[] = [
  { label: "Alice", value: 1 },
  { label: "Bob", value: 2 },
]
</script>

<template>
  <Select
    ref="selectRef"
    v-model="selectedUsers"
    :options="userOptions"
    multiple
  />
</template>
```

## Custom value mapping

When options do not follow `{ label, value }`, use `getOptionLabel` and `getOptionValue`. TypeScript infers `GenericOption` from your array:

```vue [App.vue]
<script setup lang="ts">
import { ref } from "vue"
import Select from "vue3-select-component"

type Role = { id: string; key: string }

const activeRole = ref<string | null>(null)

const roleOptions: Role[] = [
  { id: "Admin", key: "admin" },
  { id: "User", key: "user" },
  { id: "Guest", key: "guest" },
]
</script>

<template>
  <Select
    v-model="activeRole"
    :options="roleOptions"
    :get-option-label="(option) => option.id"
    :get-option-value="(option) => option.key"
    placeholder="Pick a role"
  />
</template>
```

::warning
Custom mappers weaken the link between your array type and `SelectOptionData`. Prefer typed source objects and mappers over casting options to `SelectOptionData[]`. See [Options](/guide/options#use-custom-option-shapes) for runtime behavior.
::

If inference fails, set generics explicitly:

```vue
<Select<string, Role>
  v-model="activeRole"
  :options="roleOptions"
  :get-option-label="(option) => option.id"
  :get-option-value="(option) => option.key"
/>
```

## Typed events and filters

Assembled `Select` events receive your **source** option object, not the normalized `{ label, value }` shape:

```ts
import type { AssembledSelectEmits, SelectOptionData } from "vue3-select-component"

type UserOption = SelectOptionData<number> & { username: string }

// optionSelected: [option: UserOption]
// optionDeselected: [option: UserOption | null]
type SelectEvents = AssembledSelectEmits<UserOption>
```

`filterBy` on assembled `Select` also receives the source object:

```ts
type UserOption = SelectOptionData<number> & { username: string }

type FilterBy = (
  option: UserOption,
  label: string,
  search: string,
) => boolean

const filterBy: FilterBy = (option, label, search) =>
  label.toLowerCase().includes(search.toLowerCase())
  || option.username.toLowerCase().includes(search.toLowerCase())
```

On primitives, `SelectRoot` emits `OptionValue` (not the full option object). See [Events](/api/events) for both APIs.

## Import component prop types

All public prop and emit types are exported for wrappers and design systems:

```ts
import type {
  AssembledSelectProps,
  AssembledSelectEmits,
  SelectModelValue,
  SelectOptionData,
  SelectPopoverProps,
  SelectRootProps,
  SelectRootEmits,
} from "vue3-select-component"
```

Example wrapper:

```ts
import type { AssembledSelectProps, SelectOptionData } from "vue3-select-component"

type AppSelectProps<T extends string | number> =
  AssembledSelectProps<SelectOptionData<T>, T>
```

## Primitives typing

`SelectRoot` accepts normalized `SelectOptionData<Value>[]` and types `v-model` the same way as assembled `Select`:

```vue [App.vue]
<script setup lang="ts">
import type { SelectOptionData } from "vue3-select-component"
import { ref } from "vue"
import { SelectRoot, SelectTrigger, SelectValue, SelectPopover, SelectListbox, SelectOption } from "vue3-select-component"

const selected = ref<string | null>(null)

const options: SelectOptionData<string>[] = [
  { label: "JavaScript", value: "js" },
  { label: "TypeScript", value: "ts" },
]
</script>

<template>
  <SelectRoot v-model="selected" :options="options">
    <SelectTrigger>
      <SelectValue placeholder="Pick a language" />
    </SelectTrigger>
    <SelectPopover>
      <SelectListbox>
        <SelectOption
          v-for="option in options"
          :key="option.value"
          :value="option.value"
          :label="option.label"
        />
      </SelectListbox>
    </SelectPopover>
  </SelectRoot>
</template>
```

Popover-related prop types (`SelectPopoverProps`, `SelectPopoverContentProps`, and others) are exported from the package entry point for custom compositions.
