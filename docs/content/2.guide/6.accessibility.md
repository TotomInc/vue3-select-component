Vue 3 Select Component follows the [WAI-ARIA Combobox Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/combobox/) for assistive technologies. The assembled `Select` and primitives share the same accessibility behavior.

## Overview

Built-in support includes:

- **ARIA attributes** for semantic markup (`combobox`, `listbox`, `option`)
- **Keyboard navigation** for full keyboard-only operation
- **Focus management** with `aria-activedescendant` on the searchable input
- **Screen reader support** with descriptive labels and state attributes

## Searchable vs non-searchable

The active control depends on `searchable`:

| Mode | Control element | ARIA owner |
|------|-----------------|------------|
| `searchable` (assembled default) | `SelectInput` (`role="combobox"`) | Input owns the listbox relationship |
| `searchable={false}` | `SelectTrigger` (`<button>`) | Button owns the listbox relationship |

In searchable mode, `SelectTrigger` is a layout wrapper (`<div>`). The input is the combobox. In non-searchable mode, the trigger button opens the menu and carries `aria-expanded`, `aria-haspopup`, and `aria-controls`.

Both APIs use the same primitives underneath. Pass label attributes to the control that is actually focusable.

## ARIA attributes

### Searchable combobox (`SelectInput`)

When `searchable` is enabled, the search input is the combobox control:

```html
<input
  role="combobox"
  aria-autocomplete="list"
  aria-haspopup="listbox"
  aria-expanded="true|false"
  aria-controls="v1-select-1-listbox"
  aria-activedescendant="select-option-3"
  autocomplete="off"
/>
```

| Attribute | Purpose |
|-----------|---------|
| `aria-expanded` | Menu open (`true`) or closed (`false`) |
| `aria-haspopup="listbox"` | Signals a listbox popup |
| `aria-controls` | References the listbox element ID |
| `aria-activedescendant` | Points to the keyboard-focused option (only while open) |
| `aria-autocomplete="list"` | Suggestions appear in the listbox |

### Non-searchable trigger (`SelectTrigger`)

When search is disabled, the trigger renders as a `<button>`:

```html
<button
  type="button"
  aria-expanded="true|false"
  aria-haspopup="listbox"
  aria-controls="v1-select-1-listbox"
>
```

The button is disabled with the native `disabled` attribute. `SelectInput` is not rendered in this mode.

### Listbox (`SelectListbox`)

```html
<div
  role="listbox"
  aria-labelledby="v1-select-1-input"
  aria-multiselectable="true|false"
  tabindex="-1"
>
```

| Attribute | Purpose |
|-----------|---------|
| `aria-labelledby` | References the input ID in searchable mode, or the trigger ID otherwise |
| `aria-multiselectable` | `true` when `multiple` is enabled |

### Options (`SelectOption`)

Each option renders as:

```html
<div
  role="option"
  aria-selected="true|false"
  aria-disabled="true|false"
  data-active="true|false"
>
```

| Attribute | Purpose |
|-----------|---------|
| `aria-selected` | Whether the option is in the current selection |
| `aria-disabled` | Whether the option cannot be selected |
| `data-active` | Keyboard highlight state (style with `[data-active="true"]`) |

Disabled options are skipped during keyboard navigation.

### Popover panel (`SelectPopover`)

The dropdown panel exposes visibility to assistive technologies:

```html
<div data-select-popover aria-hidden="true|false" hidden>
```

When the menu is closed, the panel is hidden and marked `aria-hidden="true"`. Auto-focus on open is prevented so focus stays on the input or trigger.

### Interactive controls

| Element | Accessible name |
|---------|-----------------|
| Clear button (`SelectClear`) | `aria-label="Clear selection"` |
| Tag remove button (`SelectTag`) | `aria-label="Remove"` |
| Leading / trailing icons | `aria-hidden="true"` (decorative) |
| Empty state (`SelectNoOptions`) | `role="status"` |

Customize tag remove labels in the `tag-remove` slot if you need option-specific text. See [Slots](/api/slots).

## Label your select

Associate the select with a visible label using standard HTML attributes. On the assembled `Select`, `id`, `aria-label`, `aria-labelledby`, `aria-describedby`, `aria-invalid`, and `aria-required` are forwarded to the active control (`SelectInput` or `SelectTrigger`).

### Assembled Select

```vue [App.vue]
<script setup lang="ts">
import { ref } from "vue"
import Select from "vue3-select-component"
import "vue3-select-component/styles"

const country = ref<string | null>(null)

const countries = [
  { label: "France", value: "FR" },
  { label: "Germany", value: "DE" },
]
</script>

<template>
  <label id="country-label" for="country-select">Country</label>
  <Select
    id="country-select"
    v-model="country"
    :options="countries"
    aria-labelledby="country-label"
    aria-required="true"
    placeholder="Select a country"
  />
</template>
```

Add helper or error text with `aria-describedby`:

```vue [App.vue]
<script setup lang="ts">
import { ref } from "vue"
import Select from "vue3-select-component"
import "vue3-select-component/styles"

const country = ref<string | null>(null)
const hasError = ref(false)

const countries = [
  { label: "France", value: "FR" },
  { label: "Germany", value: "DE" },
]
</script>

<template>
  <label id="country-label" for="country-select">Country</label>
  <p id="country-helper">Pick the country where you live.</p>
  <Select
    id="country-select"
    v-model="country"
    :options="countries"
    aria-labelledby="country-label"
    aria-describedby="country-helper"
    :aria-invalid="hasError"
    placeholder="Select a country"
  />
</template>
```

### Primitives

Pass the same attributes to the focusable primitive:

```vue [CustomSelect.vue]
<script setup lang="ts">
import { ref } from "vue"
import {
  SelectInput,
  SelectListbox,
  SelectNoOptions,
  SelectOption,
  SelectPopover,
  SelectRoot,
  SelectTrigger,
  SelectValue,
} from "vue3-select-component/primitives"
import "vue3-select-component/styles"

const model = ref<string | null>(null)

const options = [
  { label: "JavaScript", value: "js" },
  { label: "TypeScript", value: "ts" },
]
</script>

<template>
  <label id="language-label" for="language-select">Language</label>

  <div data-assembled-select>
    <SelectRoot v-model="model" :options="options" searchable>
      <SelectTrigger>
        <SelectValue placeholder="Pick a language" />
        <SelectInput
          id="language-select"
          aria-labelledby="language-label"
        />
      </SelectTrigger>

      <SelectPopover>
        <SelectListbox>
          <SelectNoOptions />
          <SelectOption
            v-for="opt in options"
            :key="opt.value"
            :value="opt.value"
            :label="opt.label"
          />
        </SelectListbox>
      </SelectPopover>
    </SelectRoot>
  </div>
</template>
```

When `searchable` is disabled, pass label attributes to `SelectTrigger` instead of `SelectInput`.

## Keyboard navigation

Keyboard handling is shared between assembled `Select` and primitives.

### When the menu is closed

| Key | Action |
|-----|--------|
| `ArrowDown` / `ArrowUp` | Open the menu and highlight the first enabled option |
| `Space` | Open the menu (trigger only, when search text is empty) |
| `Enter` | Open the menu on a focused non-searchable trigger button (native button behavior) |
| `Tab` | Move focus to the next focusable element |

In searchable mode, focusing or clicking the input also opens the menu.

### When the menu is open

| Key | Action |
|-----|--------|
| `ArrowDown` | Move to the next enabled option (wraps to the first) |
| `ArrowUp` | Move to the previous enabled option (wraps to the last) |
| `Enter` | Select the active option |
| `Space` | Select the active option when search text is empty and focus is not in the search input |
| `Escape` | Close the menu without selecting |
| `Tab` | Close the menu and move focus |
| `Backspace` | Remove the last selected tag when search is empty (multi-select only) |

Single-select closes the menu after selection by default. Multi-select keeps it open so users can pick multiple values. Override with `closeOnSelect`.

### Search behavior

When `searchable` is enabled:

- Typing filters options in real time
- The active option syncs to the first matching result
- Arrow keys move within the filtered list
- The `@search` event fires as the query changes

## Focus management

When the menu opens, the first enabled option becomes active. In searchable mode, focus stays on the input while `aria-activedescendant` points at the highlighted option.

Disabled options are skipped:

```vue [App.vue]
<script setup lang="ts">
import { ref } from "vue"
import Select from "vue3-select-component"
import "vue3-select-component/styles"

const selected = ref<string | null>(null)
</script>

<template>
  <Select
    v-model="selected"
    :options="[
      { label: 'Available', value: '1' },
      { label: 'Unavailable', value: '2', disabled: true },
    ]"
    placeholder="Pick an option"
  />
</template>
```

### Visual focus indicator

Keyboard-highlighted options use `data-active="true"`. Default assembled styles map this to a focus background. Customize with CSS variables:

```css
:root {
  --vs-option-focused-background-color: #dbeafe;
  --vs-option-focused-text-color: #18181b;
}
```

See [Styling](/guide/styling) for the full variable list and headless `data-*` hooks.

## Dismiss behavior

- Click outside the select closes the menu
- `Tab` away closes an open menu
- `Escape` closes without selecting

## Screen reader support

Assistive technologies can observe:

- Menu open/close via `aria-expanded` on the control and `aria-hidden` on the popover
- The listbox name via `aria-labelledby`
- The active option via `aria-activedescendant` while navigating
- Selection state via `aria-selected` on each option
- Disabled state via `aria-disabled`
- Empty results via `role="status"` on `SelectNoOptions`

Selected values render as visible text in `SelectValue` (single label or multi-select tags). Add `aria-label` or `aria-labelledby` on the control when the visible label alone is not enough.

## Unique IDs

Each select instance generates stable IDs for ARIA wiring:

```html
<!-- Searchable -->
<input id="v1-select-1-input" role="combobox" aria-controls="v1-select-1-listbox">
<div id="v1-select-1-listbox" role="listbox" aria-labelledby="v1-select-1-input">

<!-- Non-searchable -->
<button id="v1-select-1-trigger" aria-controls="v1-select-1-listbox">
<div id="v1-select-1-listbox" role="listbox" aria-labelledby="v1-select-1-trigger">
```

Option elements receive unique IDs for `aria-activedescendant`. Pass a custom `id` to `SelectInput` when you need a predictable input ID for `for`/`aria-labelledby` wiring.

## Test with assistive technology

Verify these scenarios during development:

1. Open and close the menu with keyboard only
2. Navigate options with arrow keys and select with Enter
3. Confirm screen readers announce expanded/collapsed state
4. Verify disabled options are skipped
5. In multi-select, confirm tags are visible and removable
6. Repeat in both searchable and non-searchable modes
7. Repeat with assembled `Select` and a custom primitives layout
