v1 is a breaking release. It adds primitives alongside the assembled `Select`. This guide covers the main changes from v0.

## Understand the new architecture

| v0 | v1 |
|----|-----|
| Single monolithic component | Assembled `Select` + primitives |
| Bundled CSS (`--vs-*` variables) | Optional default styles + headless primitives |
| Slots on `VueSelect` | Slots on individual primitives |
| `Option<T>` type | `SelectOptionData<T>` type |

The assembled `Select` covers common use cases with v1 prop names only. v0 prop aliases are **not** supported — rename them during migration.

## Rename boolean props

v1 uses clearer boolean prop names. Update every usage in your templates and script setup:

| v0 prop | v1 prop |
|---------|---------|
| `isMulti` | `multiple` |
| `isSearchable` | `searchable` |
| `isClearable` | `clearable` |
| `isDisabled` | `disabled` |
| `isLoading` | `loading` |

```vue
<!-- v0 -->
<Select :is-multi="true" :is-searchable="true" />

<!-- v1 -->
<Select multiple />
```

Primitives accept v1 names only. Assembled `Select` is searchable by default, so keep `searchable` only when you need to pass a dynamic value.

## Update imports

```ts
// v0
import VueSelect from "vue3-select-component"
import "vue3-select-component/styles"

// v1
import Select from "vue3-select-component"
import "vue3-select-component/styles"

// Primitives (new)
import { SelectRoot, SelectTrigger } from "vue3-select-component/primitives"
```

## Update bundled CSS

Keep `import "vue3-select-component/styles"` for the assembled `Select`. Default styles still expose `--vs-*` variables and target `data-*` hooks instead of v0 class names.

For custom primitive compositions, skip the stylesheet and style via `data-*` selectors. See [Styling](/guide/styling).

## Map slots to primitives

v0 slots on the monolithic component map to primitive slots:

| v0 slot | v1 equivalent |
|---------|---------------|
| `#option` | `SelectOption` default slot |
| `#value` | `SelectValue` default slot |
| `#tag` | `SelectTag` default + `remove` slots |
| `#tag-content` | `SelectTag` default slot |
| `#loading` | `SelectTrailingIcon` default slot (`loading` prop) |

The assembled `Select` exposes `icon`, `trailing-icon`, `tag-remove`, `no-options`, and `clear` slots. Enable the clear button with `clearable`. Use primitives for broader customization.

## Event payload differences

On **assembled `Select`**, events still emit source option objects (v0 behavior):

```vue
<Select @option-selected="(option) => console.log(option)" />
```

On **`SelectRoot` primitives**, events emit raw values:

```vue
<SelectRoot @option-selected="(value) => console.log(value)" />
```

Event names use camelCase in script (`@optionSelected`) and kebab-case in templates (`@option-selected`).

## Removed v0 features

These v0 props are not part of v1 yet. Use primitives to implement equivalent behavior:

| v0 prop | v1 approach |
|---------|-------------|
| `displayedOptions` | Filter `options` yourself or use custom `filterBy` |
| `isTaggable` | Handle creation in your app and add options dynamically |
| `isMenuOpen` | Control open state via primitives or custom composition |
| `hideSelectedOptions` | Use `hideSelected` on assembled `Select` or `SelectRoot` |
| `aria` prop | Apply ARIA attributes on primitive elements directly |

## Update TypeScript imports

```ts
// v0
import type { Option } from "vue3-select-component"

// v1
import type { SelectOptionData, SelectModelValue } from "vue3-select-component"
```
