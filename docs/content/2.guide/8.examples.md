Live examples for common setups. Each demo prints the current `v-model` value below the control.

## Assembled select

The default export wires up primitives with sensible defaults. Use these patterns when the built-in UX is enough.

### Clearable

Show a clear button when a value is selected.

::client-only
:example-clearable-demo

#fallback
:select-demo-fallback
::

```vue [App.vue]
<Select
  v-model="selected"
  clearable
  :options="options"
  placeholder="Select a country"
/>
```

## Primitives

Compose `SelectRoot`, `SelectTrigger`, `SelectPopover`, and the rest when you need custom layout, slots, or popover behavior. Demos use `data-assembled-select` on a wrapper so default styles apply while you own the DOM tree.

### Full composition

Wire every trigger and listbox primitive yourself. Add a leading icon, search input, clear button, and a custom trailing icon.

::client-only
:example-primitive-composed-demo

#fallback
:select-demo-fallback
::

```vue [CustomSelect.vue]
<SelectRoot v-model="model" searchable clearable>
  <SelectTrigger>
    <SelectIcon><!-- globe icon --></SelectIcon>
    <SelectValue placeholder="Select a country" />
    <SelectInput />
    <SelectTrailingIcon>
      <template #default="{ open }">
        {{ open ? "▲" : "▼" }}
      </template>
    </SelectTrailingIcon>
    <SelectClear />
  </SelectTrigger>

  <SelectPopover>
    <SelectListbox>
      <SelectNoOptions />
      <SelectOption
        v-for="option in options"
        :key="option.value"
        v-bind="option"
      />
    </SelectListbox>
  </SelectPopover>
</SelectRoot>
```

### Multi-select tags

Customize tag remove buttons and option checkmarks in multi-select mode.

::client-only
:example-primitive-multi-tags-demo

#fallback
:select-demo-fallback
::

```vue [CustomSelect.vue]
<SelectRoot v-model="model" multiple searchable>
  <SelectTrigger>
    <SelectValue placeholder="Select countries">
      <template #tag-remove="{ label }">
        <span :aria-label="`Remove ${label}`">×</span>
      </template>
    </SelectValue>
    <SelectInput />
    <SelectTrailingIcon />
  </SelectTrigger>

  <SelectPopover>
    <SelectListbox>
      <SelectOption v-for="opt in options" :key="opt.value" v-bind="opt">
        <template #checkmark>✓</template>
      </SelectOption>
    </SelectListbox>
  </SelectPopover>
</SelectRoot>
```

### Create item

Let users add a value from the search query. Handle `@create` to push into your options list and update `v-model`.

::client-only
:example-primitive-create-item-demo

#fallback
:select-demo-fallback
::

```vue [CustomSelect.vue]
<SelectRoot
  v-model="model"
  searchable
  create-item
  @create="onCreate"
>
  <SelectPopover>
    <SelectListbox>
      <SelectOption v-for="item in items" :key="item" :value="item" :label="item" />
      <SelectCreateItem>
        <template #default="{ searchValue }">
          Add "{{ searchValue }}"
        </template>
      </SelectCreateItem>
    </SelectListbox>
  </SelectPopover>
</SelectRoot>
```

### Filter by extra fields

Pass a custom `filterBy` to `SelectRoot` and render extra metadata in each `SelectOption` slot.

::client-only
:example-primitive-region-filter-demo

#fallback
:select-demo-fallback
::

```vue [CustomSelect.vue]
<SelectRoot
  v-model="model"
  searchable
  :filter-by="(option, search) =>
    option.label.toLowerCase().includes(search.toLowerCase())
    || getRegion(option.value).toLowerCase().includes(search.toLowerCase())"
>
  <SelectOption
    v-for="country in countries"
    :key="country.code"
    :value="country.code"
    :label="country.name"
  >
    <span>{{ country.name }}</span>
    <span>{{ country.region }}</span>
  </SelectOption>
</SelectRoot>
```
