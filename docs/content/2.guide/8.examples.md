Try these live examples for everyday `Select` setups. Each demo shows the current `v-model` value below the control.

## Clearable

Show a clear button when a value is selected.

::client-only
:example-clearable-demo

#fallback
:select-demo-fallback
::

```vue [App.vue]
<Select
  v-model="selected"
  clearable
  :options="options"
  placeholder="Select a country"
/>
```

## Non-searchable

Turn off the search input for a button-only select.

::client-only
:example-non-searchable-demo

#fallback
:select-demo-fallback
::

```vue [App.vue]
<Select
  v-model="selected"
  :searchable="false"
  :options="options"
  placeholder="Select an option"
/>
```

## Loading

Expose a loading state on the trigger while options are fetched.

::client-only
:example-loading-demo

#fallback
:select-demo-fallback
::

```vue [App.vue]
<Select
  v-model="selected"
  :loading="isLoading"
  :options="options"
/>
```

## Hide selected options

In multi-select mode, remove picked options from the dropdown while keeping them as tags in the trigger.

::client-only
:example-hide-selected-demo

#fallback
:select-demo-fallback
::

```vue [App.vue]
<Select
  v-model="selected"
  multiple
  hide-selected
  :options="options"
/>
```

## Disabled options

Mark individual options as disabled so they cannot be selected.

::client-only
:example-disabled-options-demo

#fallback
:select-demo-fallback
::

```vue [App.vue]
<Select
  v-model="selected"
  :options="[
    { label: 'France', value: 'FR' },
    { label: 'Spain', value: 'ES', disabled: true },
    { label: 'Germany', value: 'DE' },
  ]"
/>
```

## Close on select

By default, the menu stays open in multi-select mode. Set `close-on-select` to dismiss it after each pick.

::client-only
:example-close-on-select-demo

#fallback
:select-demo-fallback
::

```vue [App.vue]
<Select
  v-model="selected"
  multiple
  close-on-select
  :options="options"
/>
```
