## Assembled Select

The default export from `vue3-select-component`.

::note
v0 boolean prop names (`isMulti`, `isSearchable`, etc.) are not supported. Rename them to v1 names when upgrading. See the [migration guide](/guide/migration).
::

### v-model

| | |
|---|---|
| **Type** | `string \| number \| null` (single) or `(string \| number)[]` (multi) |
| **Required** | No |
| **Default** | `null` (single) / `[]` (multi) |

The selected value(s). Type must match `option.value`. See [TypeScript](/guide/typescript).

### options

| | |
|---|---|
| **Type** | `GenericOption[]` |
| **Required** | Yes |

List of options. Each option needs `label` and `value` unless custom mappers are provided.

```ts
type SelectOptionData<Value extends string | number> = {
  label: string
  value: Value
  disabled?: boolean
}
```

### placeholder

| | |
|---|---|
| **Type** | `string` |
| **Default** | `undefined` |

Text shown when no option is selected. Passed to `SelectValue`.

### multiple

| | |
|---|---|
| **Type** | `boolean` |
| **Default** | `false` |

Allow multiple selections. `v-model` must be an array.

### closeOnSelect

| | |
|---|---|
| **Type** | `boolean \| null` |
| **Default** | `null` |

Control whether the menu closes after a selection. When `null` (default), the menu closes on single-select and stays open on multi-select. Set to `true` or `false` to override.

### hideSelected

| | |
|---|---|
| **Type** | `boolean` |
| **Default** | `false` |

When `multiple` is enabled, remove already-selected options from the dropdown list. Selected values remain as tags in the trigger. Has no effect in single-select mode.

```vue
<Select
  v-model="selected"
  multiple
  hide-selected
  :options="options"
/>
```

### searchable

| | |
|---|---|
| **Type** | `boolean` |
| **Default** | `true` |

Show a search input in the trigger to filter options. Set `:searchable="false"` to disable it.

### clearable

| | |
|---|---|
| **Type** | `boolean` |
| **Default** | `false` |

Show a clear button when a value is selected.

### disabled

| | |
|---|---|
| **Type** | `boolean` |
| **Default** | `false` |

Disable all interaction.

### loading

| | |
|---|---|
| **Type** | `boolean` |
| **Default** | `false` |

Expose loading state to `SelectTrailingIcon` via `data-loading`.

### Popover props

The assembled `Select` forwards [reka-ui](https://reka-ui.com/docs/components/popover) popover props to the internal `SelectPopover`. Use them to control placement, collision behavior, modality, and teleport deferral.

| Prop | Type | Default | Maps to |
|------|------|---------|---------|
| `teleport` | `boolean \| string \| HTMLElement` | `true` | `PopoverPortal` (`to` / `disabled`) |
| `defer` | `boolean` | `undefined` | `PopoverPortal` |
| `modal` | `boolean` | `undefined` | `PopoverRoot` |
| `side` | `'top' \| 'right' \| 'bottom' \| 'left'` | `'bottom'` | `PopoverContent` |
| `sideOffset` | `number` | `4` | `PopoverContent` |
| `sideFlip` | `boolean` | `undefined` | `PopoverContent` |
| `align` | `'start' \| 'center' \| 'end'` | `'start'` | `PopoverContent` |
| `alignOffset` | `number` | `undefined` | `PopoverContent` |
| `alignFlip` | `boolean` | `undefined` | `PopoverContent` |
| `avoidCollisions` | `boolean` | `true` | `PopoverContent` |
| `collisionBoundary` | `Element \| null \| Element[]` | `undefined` | `PopoverContent` |
| `collisionPadding` | `number \| Partial<Record<Side, number>>` | `undefined` | `PopoverContent` |
| `arrowPadding` | `number` | `undefined` | `PopoverContent` |
| `hideShiftedArrow` | `boolean` | `undefined` | `PopoverContent` |
| `sticky` | `'partial' \| 'always'` | `undefined` | `PopoverContent` |
| `hideWhenDetached` | `boolean` | `undefined` | `PopoverContent` |
| `positionStrategy` | `'absolute' \| 'fixed'` | `undefined` | `PopoverContent` |
| `updatePositionStrategy` | `'optimized' \| 'always'` | `undefined` | `PopoverContent` |
| `disableUpdateOnLayoutShift` | `boolean` | `undefined` | `PopoverContent` |
| `prioritizePosition` | `boolean` | `undefined` | `PopoverContent` |
| `disableOutsidePointerEvents` | `boolean` | `undefined` | `PopoverContent` |

`teleport` controls where the panel is rendered. `true` (default) teleports to `body`. A string is a CSS selector target. An `HTMLElement` teleports to that element. Set to `false` to render inline.

`open`, anchor `reference`, and `forceMount` are managed internally and are not exposed.

```vue
<Select
  v-model="value"
  :options="options"
  side="top"
  :side-offset="8"
  align="end"
/>
```

### getOptionLabel

| | |
|---|---|
| **Type** | `(option: GenericOption) => string` |
| **Default** | `option => option.label` |

Extract the display label from a custom option shape.

### getOptionValue

| | |
|---|---|
| **Type** | `(option: GenericOption) => string \| number` |
| **Default** | `option => option.value` |

Extract the value from a custom option shape.

### filterBy

| | |
|---|---|
| **Type** | `(option, label, search) => boolean` |
| **Default** | Case-insensitive label substring match |

Custom filter for searchable mode. Receives the **source** option object.

## SelectRoot

Props for the primitive root component (`vue3-select-component/primitives`).

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `v-model` | `SelectModelValue<Value>` | `null` | Selected value(s) |
| `options` | `SelectOptionData<Value>[]` | `[]` | Normalized options array |
| `multiple` | `boolean` | `false` | Multi-select mode |
| `searchable` | `boolean` | `false` | Enable trigger search input |
| `clearable` | `boolean` | `false` | Show clear button |
| `disabled` | `boolean` | `false` | Disable interaction |
| `loading` | `boolean` | `false` | Loading state |
| `closeOnSelect` | `boolean \| null` | `null` | Close menu after selection (`null` = auto) |
| `hideSelected` | `boolean` | `false` | Hide selected options from the list in multi-select |
| `filterBy` | `FilterByFn<Value>` | Label substring | Custom filter on registered options |

### SelectPopover

`SelectPopover` forwards [reka-ui](https://reka-ui.com/docs/components/popover) `PopoverRoot`, `PopoverPortal`, and `PopoverContent` props. See the assembled [Popover props](#popover-props) table for the full list.

| Prop | Type | Default |
|------|------|---------|
| `teleport` | `boolean \| string \| HTMLElement` | `true` |
| `side` | `'top' \| 'right' \| 'bottom' \| 'left'` | `'bottom'` |
| `align` | `'start' \| 'center' \| 'end'` | `'start'` |
| `sideOffset` | `number` | `4` |
| `avoidCollisions` | `boolean` | `true` |

All other supported popover props from the table above are also available on `SelectPopover`. Content emits (`escapeKeyDown`, `pointerDownOutside`, `focusOutside`, `interactOutside`, `openAutoFocus`, `closeAutoFocus`) can be listened to on `SelectPopover`.

`SelectTrigger`, `SelectListbox`, `SelectClear`, `SelectIcon`, `SelectTrailingIcon`, and `SelectNoOptions` accept no props. `SelectInput` accepts optional `id` and `placeholder` props.
