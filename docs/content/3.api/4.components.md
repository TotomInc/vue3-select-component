All primitives are exported from `vue3-select-component/primitives` and must be used inside `SelectRoot`.

## SelectRoot

State provider and context source. Manages `v-model`, open state, search, filtering, keyboard navigation, and option registration.

| | |
|---|---|
| **Props** | See [Props](/api/props#selectroot) |
| **Emits** | See [Events](/api/events#selectroot-events) |
| **Slots** | `default`, trigger and popover composition |
| **DOM** | `data-select-root` |

## SelectTrigger

The interactive button that toggles the menu. Handles click and keyboard events.

| | |
|---|---|
| **Props** | None |
| **Slots** | `default`, typically `SelectValue`, `SelectIndicator`, `SelectClear` |
| **DOM** | `data-select-trigger` |
| **ARIA** | `aria-expanded`, `aria-haspopup="listbox"`, `aria-controls`, `role="combobox"` when searchable |

## SelectValue

Displays the current selection or placeholder.

| | |
|---|---|
| **Props** | `placeholder?: string` |
| **Slots** | `default`, `{ selectedOptions }`, `tag-remove`, `{ label, value }` |
| **DOM** | `data-select-value`, `data-empty` |

## SelectInput

Search input rendered when `searchable` is enabled. Lives inside `SelectPopover` in the assembled layout.

| | |
|---|---|
| **Props** | None |
| **DOM** | `data-select-input` |
| **Notes** | `autocomplete="off"`, bound to context search value |

## SelectPopover

Dropdown panel container built on reka-ui `Popover`. Supports teleporting and forwards reka-ui popover props for placement, collision handling, and modality. See [Popover props](/api/props#popover-props).

| | |
|---|---|
| **Props** | `teleport`, `side`, `align`, `sideOffset`, `modal`, `defer`, and other [reka-ui `PopoverContent` props](/api/props#popover-props) |
| **Emits** | `escapeKeyDown`, `pointerDownOutside`, `focusOutside`, `interactOutside`, `openAutoFocus`, `closeAutoFocus` |
| **Slots** | `default`, input and listbox |
| **DOM** | `data-select-popover` |
| **ARIA** | `aria-hidden` when closed |

## SelectListbox

Container for options with `role="listbox"`.

| | |
|---|---|
| **Props** | None |
| **Slots** | `default`, `SelectOption` and `SelectNoOptions` |
| **DOM** | `data-select-listbox` |
| **ARIA** | `role="listbox"`, `aria-multiselectable`, `tabindex="-1"` |

## SelectOption

A single selectable option. Registers with context on mount.

| | |
|---|---|
| **Props** | `value`, `label`, `disabled?` |
| **Slots** | `default`, `checkmark` |
| **DOM** | `data-select-option`, `data-select-option-label`, `data-select-option-checkmark`, `data-value`, `data-active` |
| **ARIA** | `role="option"`, `aria-selected`, `aria-disabled` |

Hidden when filtered out by search or when `hideSelected` removes selected values in multi-select. In multi-select mode, selected options show a default checkmark icon on the right (overridable via the `checkmark` slot). Click calls `context.select(value)`.

## SelectTag

Chip for a selected value in multi-select mode.

| | |
|---|---|
| **Props** | `value`, `label` |
| **Slots** | `default`, `{ label, value }`, `remove`, `{ label, value }` |
| **DOM** | `data-select-tag`, `data-select-tag-remove` |

## SelectClear

Button to reset the selection.

| | |
|---|---|
| **Props** | None |
| **Slots** | `default` |
| **DOM** | `data-select-clear` |
| **ARIA** | `aria-label="Clear selection"` |

Visible when `clearable && hasValue && !disabled`.

## SelectIndicator

Decorative indicator for open/loading state.

| | |
|---|---|
| **Props** | None |
| **Slots** | `default`, `{ open, loading }` |
| **DOM** | `data-select-indicator`, `data-open`, `data-loading` |
| **ARIA** | `aria-hidden="true"` |

## SelectNoOptions

Empty state when no options match the filter.

| | |
|---|---|
| **Props** | None |
| **Slots** | `default`, `{ searchValue }` |
| **DOM** | `data-select-no-options` |
| **ARIA** | `role="status"` |

## Assembled Select

Fixed composition of all primitives. See [Usage](/getting-started/usage) and [Props](/api/props#assembled-select).

| | |
|---|---|
| **Import** | `import Select from "vue3-select-component"` |
| **Slots** | `tag-remove`, `{ label, value }` (multi-select), `no-options`, `{ searchValue }`, `clear` |
| **Props** | Includes `clearable` to show the clear button |
| **DOM** | `data-assembled-select` on root |

**Default structure:**

```
SelectRoot
├── SelectTrigger
│   ├── SelectValue
│   ├── SelectIndicator
│   └── SelectClear
└── SelectPopover
    ├── SelectInput
    └── SelectListbox
        ├── SelectNoOptions
        └── SelectOption (per option)
```
